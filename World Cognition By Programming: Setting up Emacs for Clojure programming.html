Setting up Emacs for Clojure programming

When starting my first Clojure journey recently, I faced the problem of setting up Emacs for Clojure programming. It can seem cumbersome to do it right if you aren't aware of some pitfalls, and many tutorials and answers at stackoverflow are outdated. So, I leave simple up to date instructions here intended for complete Emacs beginners (you must have at least read the built-in tutorial available on C-h t).

Why Emacs

Personally, I find Emacs the most suited editor for Clojure (far beyond Eclipse, IntelliJ plugins). It supports Lisps very well through being itself written and extended (mostly) in Lisp dialect. Secondly, I loved its minimalistic and fully configurable interface, so now I find myself trying to reduce the visual clutter when using Eclipse or IntelliJ by hiding tool-bars, panels, menus and so on. And finally: you will never overgrow it. Using Emacs is a perpetual process of discovering something new and training your brains by memorizing all those key-bindings. :) And you needn't to use arrows keys anymore (you will really hate them after becoming accustomed with Emacs key-map). In these instructions I omitted most of unrelated to Clojure programming Emacs tricks (but left some useful ones) aiming to end up with simplest but still working configuration to start coding (e.g. by reading some good book like this one). Anyone is welcome to extend his config later. I just want not to mix up general Emacs configuration with Clojure-related one.

Choosing version

This tutorial aims to be OS-agnostic so the setup will work on Linux, OSX, Windows without any changes. Just download the appropriate Emacs 24 for your OS and you are ready. It's crucial to use version 24 otherwise instructions won't work. For OSX users I recommend this version and not Aquamacs.

Leiningen 2

Download and install Leiningen 2 - a tool for building Clojure and Java code compatible with Maven. Issue 'lein version' in terminal to make sure it works: 
> lein version
Leiningen 2.0.0-preview7 on Java 1.6.0_29 Java HotSpot(TM) 64-Bit Server VM
Leiningen is required for REPL functionality.

.emacs configuration

Locate .emacs file in your home directory or create a new one if it doesn't exist. It'll contain all the configuration. First, open it in Emacs and add Melpa repository: 
;; Package.el customization
(package-initialize)
(add-to-list 'package-archives
             '("melpa" . "http://melpa.milkbox.net/packages/") t)
Evaluate each top-level expression (there are two here) by moving cursor at each expression end (after closing parenthesis) and press C-x C-e. Alternatively, you can just restart Emacs to make it evaluate those itself. Next, issue M-x package-list-packages and select the following packages from the list (just press 'i' having cursor positioned on appropriate lines):
  • ac-nrepl
  • auto-complete
  • clojure-mode
  • nrepl
  • paredit
  • popup
  • rainbow-delimiters
To apply changes press 'x' and confirm.

rainbow-delimiters is an Emacs mode to colorize parentheses which is very handy for Lisps (and visually appealing). Enable it globally by adding: 
;; rainbow delimiters
(global-rainbow-delimiters-mode)
As usually, evaluate it with C-x C-e or restart Emacs. You'll see now parentheses highlighted in your .emacs file!

paredit is an Emacs mode which helps (and sometimes prevents (: ) editing parentheses-structured data and code (which is data in Lisps anyway). The best way to get feet wet is to try key combinations from the cheatsheet. It's really worse learning, but for beginners it can be quite annoying so I've bound 'f7' key for that special purpose. Another not so powerful but simpler alternative with less learning curve is autopair, but try paredit first! Enable it when editing Clojure source and in REPL: 
;; paredit
(add-hook 'clojure-mode-hook 'paredit-mode)
(add-hook 'nrepl-mode-hook 'paredit-mode)
(global-set-key [f7] 'paredit-mode)
Evaluate and... that's all! You can now edit .clj files having syntax highlighting, nice parentheses coloring, which are automatically inserted.

REPL

Beyond syntax highlighting and auto-indentation what is really expected is REPL and that's where Emacs shines at its best. nREPL elisp client is used just for that purpose. To configure it add the following lines (they are just taken from github page) to the config: 
;; nrepl
(add-hook 'nrepl-interaction-mode-hook 'nrepl-turn-on-eldoc-mode)
(setq nrepl-popup-stacktraces nil)
(add-to-list 'same-window-buffer-names "*nrepl*")
(add-hook 'nrepl-mode-hook 'paredit-mode)
nrepl-jack-in command is used to start interactive REPL session; one might prefer to bind it to some key: 
(global-set-key [f9] 'nrepl-jack-in)
So, issue 'f9' or M-x nrepl-jack-in. REPL session starts. nREPL is another mode which key-bindings are worst memorizing. For example, position cursor on some function usage, like println, and press C-c C-d. Documentation window appears. To load some buffer (perhaps, a *.clj file) into REPL press C-c C-l; to evaluate a single expression - our old good friend C-x C-e comes handy. To clear the REPL issue C-c M-o.

Auto-complete and documentation popup

auto-complete, ac-nrepl and popup packages provide auto-complete facilities while editing Clojure source and in REPL. Enable them: 
;; Auto complete
(require 'auto-complete-config)
(ac-config-default)
(define-key ac-completing-map "\M-/" 'ac-stop) ; use M-/ to stop completion

;; ac-nrepl
(require 'ac-nrepl)
(add-hook 'nrepl-mode-hook 'ac-nrepl-setup)
(add-hook 'nrepl-interaction-mode-hook 'ac-nrepl-setup)
(eval-after-load "auto-complete" '(add-to-list 'ac-modes 'nrepl-mode))

To navigate the variants up and down use M-p and M-n keys; M-/ to stop completion. You can find more info on these packages on their respective github pages.

Not letting you go blind

To make Emacs colors more visually appealing I recommend to install color-theme-solarized (using package manager as we did for other modes) and load it on start-up: 
;; theme
(load-theme 'solarized-light t)

Going further

After you have basic Emacs setup working you might be tempted to configure it further, so at this step I recommend to take a look at:
  • desktop-save-mode - allows saving and restoring open files upon Emacs restart (built-in);
  • ido-mode - a nicer buffer switching (build-in);
  • recentf-mode - adds recent files list to menu (built-in);
  • show-paren-mode - highlights matching parentheses and braces (built-in);
  • auto-complete and ac-nrepl modes - auto complete with nREPL integration (available in Melpa repository);
You may also wish to make Emacs start with less information, disable backups, hide toolbar, and so on: 
(custom-set-variables
 ;; custom-set-variables was added by Custom.
 ;; If you edit it by hand, you could mess it up, so be careful.
 ;; Your init file should contain only one such instance.
 ;; If there is more than one, they won't work right.
 '(auto-save-default nil)
 '(backup-inhibited t t)
 '(cursor-type 'bar t)
 '(column-number-mode t)
 '(delete-selection-mode t)
 '(inhibit-startup-screen t)
 '(initial-scratch-message nil)
 '(tool-bar-mode nil))
Just add this to your .emacs or put those inner elements inside your custom-set-variables section if your config already contains one.
